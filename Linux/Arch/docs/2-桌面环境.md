# 图形化相关包

图形化的构建分为以下几层(由低到高):
1. 物理硬件层: 显卡
2. 内核驱动层: 显卡的驱动程序
3. 用户态驱动层(API指令对接GPU指令的实现): Mesa(开源API实现库集合) / NVIDIA-Utils(英伟达独家)
4. API接口层: 图形(Vulkan/OpenGL), GPU计算(OpenCL), 编解码(VAAPI)
5. 应用层: 各种应用程序和框架(如GStreamer)

另外需要安装内核头文件让驱动正常编译工作.  
某些程序(Steam/Wine)需要额外安装32位的库(lib32-xxx).  
多显卡支持也需要额外的包.  
装完后建议重启一次.

```bash
# 1. 内核头文件: 确保驱动模块正确编译
# 需要对应内核版本：linux-headers, linux-zen-headers, linux-lts-headers
pacman -S --needed linux-headers

# 2. 内核驱动层

# 2.1 Nvidia
# 查看显卡family: https://nouveau.freedesktop.org/CodeNames.html
# 查驱动程序: https://wiki.archlinux.org/title/NVIDIA
# 不同显卡使用不同, 建议用dkms版(内核变更会重新编译驱动), 具体看wiki
# pacman -S --needed nvidia-open-dkms # 较新的N卡

# 2.2 AMD
# https://wiki.archlinux.org/title/AMDGPU
# 驱动(linux-firmware-amdgpu)已包含于linux-firmware, 不用特意装

# 2.3 Intel
# https://wiki.archlinux.org/title/Intel_graphics
# 驱动(linux-firmware-intel)已包含于linux-firmware, 不用特意装

# 3. 用户态驱动层 (API 调用 -> GPU 指令)

# 3.1 AMD (使用mesa)
pacman -S mesa lib32-mesa
# 3.2 Intel (同样使用mesa)
# pacman -S mesa lib32-mesa
# Mesa 集合包含: OpenGL实现 / Vulkan驱动 / VAAPI实现 / OpenCL开源实现

# 3.3 NVIDIA
# NVIDIA 不用 Mesa, 用独家实现, 包含的实现: OpenGL, Vulkan, 基础 GPU 支持库
pacman -S nvidia-utils lib32-nvidia-utils

# 4. API 接口层

# 4.1 图形 API (OpenGL / Vulkan)
# 4.1.1 AMD
# xf86-video-amdgpu 是 X11 环境下的 2D 驱动, vulkan-headers非必需
pacman -S vulkan-radeon lib32-vulkan-radeon vulkan-headers xf86-video-amdgpu
# 4.1.2 Intel
# pacman -S vulkan-intel lib32-vulkan-intel vulkan-headers
# 4.1.3 NVIDIA
# Vulkan 驱动已由 nvidia-utils 提供
# pacman -S vulkan-icd-loader lib32-vulkan-icd-loader vulkan-headers

# 4.2 GPU 计算 API (OpenCL)
# https://wiki.archlinux.org/title/General-purpose_computing_on_graphics_processing_units#Runtime
# 4.2.1 AMD (开源通用版, 非最佳性能版)
pacman -S opencl-mesa lib32-opencl-mesa opencl-icd-loader lib32-opencl-icd-loader
# 4.2.2 Intel (同AMD)
# 4.2.3 NVIDIA
# pacman -S opencl-nvidia lib32-opencl-nvidia opencl-icd-loader lib32-opencl-icd-loader

# 4.3 视频编解码 API (VAAPI)
# https://wiki.archlinux.org/title/Hardware_video_acceleration

# 4.3.1 AMD
# 已由 Mesa 提供, 无需额外安装

# 4.3.2 Intel
# pacman -S intel-media-driver # 新架构(Broadwell以后), 其他看wiki

# 4.3.3 NVIDIA
# 由nvidia-utils和libva-nvidia-driver提供
# 可考虑替换为nvidia-vaapi-driver, 据wiki说功耗更低(archlinuxcn有)
# pacman -S libva-nvidia-driver

# 4.3.4 可选装验证工具
pacman -S libva-utils
vainfo # 验证命令(重启后执行)
# 多显卡用户要改环境变量指定显卡验证:
# LIBVA_DRIVER_NAME=xxx vainfo # xxx改成 nvidia / radeonsi

# 5. 应用层框架
# GStreamer系列插件
pacman -S gst-plugin-va # va是VAAPI插件, 一些桌面软件用到

# 6. 多显卡(未验证)
# 需要装 vulkan-mesa-layers 以支持 vk-device-select
# 包: vulkan-mesa-layers lib32-vulkan-mesa-layers
# 如果有N卡, 装 nvidia-prime switcheroo-control 且要配置GSK_RENDERER
# 需要启用 nvidia-powerd switcheroo-control 服务
# 
# 关于混合模式下用独显运行
# https://wiki.archlinuxcn.org/wiki/混合图形技术
# https://wiki.archlinuxcn.org/wiki/PRIME
# https://wiki.archlinuxcn.org/wiki/NVIDIA_Optimus
# KDE桌面环境: 开始菜单右键编辑应用程序在高级页面设置用独显运行
# 其他看文档
# 
# 双显卡的时候, 确保电源管理正确关闭独立显卡
# https://github.com/Askannz/optimus-manager/wiki/A-guide--to-power-management-options
# 
# 关于显卡切换
# KDE 桌面可以用envycontrol 和 Optimus GPU Switcher.
# 目前Wayland下不完善, 只能从混合模式切换到核显模式. 独显直连需要手动进BIOS调整
# 具体看文档
```

# 修改键盘映射

```bash
# xremap 方案
# 以下步骤适用于KDE, 其他情况参考官方文档
yay -S xremap-kde-bin
# 1. xremap 默认需要 sudo 权限, 通过下面步骤让其免去sudo
# 1.1. 启用uinput模块
lsmod | grep uinput # 检查模块是否已加载
echo uinput | sudo tee /etc/modules-load.d/uinput.conf # 如果为空, 创建模块加载文件
# 1.2. 添加 udev 权限规则(让普通用户也能访问 /dev/uinput)
echo 'KERNEL=="uinput", GROUP="input", TAG+="uaccess"' | sudo tee /etc/udev/rules.d/99-input.rules
# 1.3. 把当前用户加入 input 组
sudo gpasswd -a $USER input

# 2. 开机自启 (无需执行, 配置文件通过dotfiles管理)
# 2.1. 可通过 systemctl 自启动
# 编辑 ~/.config/systemd/user/xremap.service
# systemctl --user start xremap
# systemctl --user enable xremap
# 2.2. 官方更推荐用 autostart desktop 确保正常启动
# systemd --user 服务可能太早启动, 导致 xremap 无法检测到窗口管理器
# 编辑 ~/.config/autostart/xremap.desktop
```

输入法使用异常的解决办法

```bash
# 原因: Wayland 与 X11 在输入法协议和应用适配层面的实现差异
# 可能导致: 输入漏字; 删不掉输入框最后一个字符; 输入卡顿延迟; 删除输入框字符时误删右侧已输入内容; 输入法不可用
# 解决方式: 增加运行参数. 但要避免全局设置输入法模块, 仅对存在问题的应用单独添加运行参数.
# 说明:
# 基于Wayland运行的图形化环境(KDE, HyprLand, Niri, Sway...),
# 若全局设置 GTK/QT IM_MODULE, 可能导致输入法候选框闪烁和位置错误等问题,
# 但若不设置, 又会在 XWayland 或 Electron 应用中出现输入异常.
# 因此仅在全局设置基本语言参数, 对存在问题的应用单独指定输入法相关环境变量.
#
# 1. 修改全局基本语言参数
# 修改locale的 LANG 为英文, LC_MESSAGES 为中文: 保持中文界面同时, 避免中文locale导致的输入法异常
# 修改 ~/.config/environment.d/ 下的变量和 KDE Plasma 语言配置(~/.config/plasma-localerc)
# 写入这两个环境变量
# LANG=en_US.UTF-8
# LC_MESSAGES=zh_CN.UTF-8
# 若使用其他Wayland合成器(比如Niri), 需要在其窗口管理器配置中设置这两个参数
#
# 2. 针对存在输入异常的应用单独设置启动参数
# 2.1 对于 Qt / GTK 应用，在启动时添加环境变量: GTK_IM_MODULE=fcitx QT_IM_MODULE=fcitx
# 2.2 对于 Chromium 和 Electron 应用
#      可以先加: --ozone-platform=wayland
#      不行再加: --enable-features=UseOzonePlatform --ozone-platform=wayland --enable-wayland-ime
# 2.3 有些软件版本支持不好, 需要切换版本: --wayland-text-input-version=3 或者 2
# 2.4 有些软件 Wayland 支持不好, 尝试换到 XWayland: --ozone-platform=x11
# 2.5 有些软件locale的LC_CTYPE变量为英文时可能无法输入中文, 此时单独设置中文LC_CTYPE变量
```

2. 右键 Tampermonkey, 保存以下内容为用户脚本

```js
// ==UserScript==
// @name         双击右键菜单
// @namespace    http://tampermonkey.net/
// @version      2025-07-05
// @description  只有在双击右键时才会显示浏览器原生右键菜单
// @author       xiaochen0517
// @match        *://*/*
// @icon         https://www.google.com/s2/favicons?sz=64&domain=greasyfork.org
// @grant        none
// ==/UserScript==
(function () {
  "use strict";
  // 用于记录右键点击的时间戳
  let lastRightClick = 0;
  const doubleClickSpeed = 300; // 双击判定的时间间隔（毫秒）

  // 监听右键点击事件
  document.addEventListener(
    "contextmenu",
    function (event) {
      const currentTime = new Date().getTime();
      const timeDiff = currentTime - lastRightClick;
      // 如果时间间隔小于设定值，认为是双击右键
      if (timeDiff < doubleClickSpeed) {
        // 双击右键，允许显示原生菜单
        lastRightClick = 0; // 重置时间戳
        return true;
      } else {
        // 单击右键，阻止显示原生菜单
        event.preventDefault();
        lastRightClick = currentTime;
        return false;
      }
    },
    true
  );
})();
```

docker-compose.yml内容

```yml
services:
  xunlei:
    container_name: xunlei
    image: cnk3x/xunlei:beta
    restart: no # 想开机启动改为 unless-stopped
    hostname: arch-xunlei
    cap_add:
      - SYS_ADMIN
    ports:
      # - "2345:2345"
      - "127.0.0.1:2345:2345" # 仅本机, 省掉密码
    network_mode: bridge
    environment:
      - XL_UID=1000 # 改成uid数值
      - XL_GID=998  # 改成gid数值
      - XL_PREVENT_UPDATE=false
      - XL_DASHBOARD_USERNAME=
      - XL_DASHBOARD_PASSWORD=
    volumes:
      - /home/username/Downloads/xunlei:/xunlei/downloads # 改用户名
      - ./data:/xunlei/data
      - ./cache:/xunlei/var/packages/pan-xunlei-com
```
