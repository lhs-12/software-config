# Mise

[mise](https://mise.jdx.dev/)（发音 "meez"）是多语言开发环境管理器, 管理工具版本、环境变量和任务. 由 jdx 开发, 支持 Node.js、Java、Python 等运行时.

## 安装与初始化

```bash
curl https://mise.run | sh # 原生安装, 不用包管理器

# Shell 激活 (在 .bashrc 或 fish config 中添加以下一行)
eval "$(mise activate bash)"           # Bash
mise activate fish | source            # Fish

# 信任 stow 软链接, 使配置生效
mise trust --all ~/.config/mise

# 自动补全 (Bash)
mise completion bash > /etc/bash_completion.d/mise
```

## 配置文件

全局配置: `~/.config/mise/config.toml` (项目 dotfiles 通过 stow 软链接)

```toml
[tools]       # 工具及版本
[env]         # 环境变量
[tasks.*]     # 任务定义
[settings]    # mise 行为设置
```

项目级配置: 项目目录下的 `mise.toml`, 会与全局配置层级合并 (项目优先).

idiomatic version file: 在 `[settings]` 中启用后, mise 可读取 `.node-version`, `.python-version` 等文件.

```toml
[settings]
idiomatic_version_file_enable_tools = ["python", "node", "pnpm", "java"]
```

## 工具管理

| 操作           | 命令                               | 说明                          |
| -------------- | ---------------------------------- | ----------------------------- |
| 搜索工具       | `mise registry [关键词]`           | 查看支持哪些工具              |
| 查看可用版本   | `mise ls-remote node`              | 远程版本列表                  |
| 安装并激活     | `mise use -g node@lts`             | 安装 + 写入全局配置           |
| 仅安装         | `mise install node@lts`            | 安装但不激活                  |
| 查看已安装     | `mise ls`                          | 列出所有工具及版本            |
| 查看过期       | `mise ls --outdated`               | 查看可升级的工具              |
| 升级工具       | `mise upgrade`                     | 升级所有工具                  |
| 交互式升级     | `mise upgrade --interactive`       | 选择性升级                    |
| 升级+改版本号  | `mise upgrade --bump`              | 同时更新 mise.toml 中的版本号 |
| 卸载工具       | `mise uninstall node@20`           | 移除特定版本                  |
| 移除激活       | `mise use --global --remove node`  | 从配置中移除                  |
| 查看安装路径   | `mise where node@lts`              | 工具安装目录                  |
| 临时执行       | `mise exec node@22 -- node app.js` | 用指定版本一次性运行          |
| 更新 mise 自身 | `mise self-update`                 | 更新 mise CLI                 |

## npm 后端 — 用 mise 安装全局 npm 包

mise 支持 `npm:` 前缀直接从 npm registry 安装工具, 等价于传统的 `npm install -g`, 但由 mise 统一管理版本和 PATH. 当 aube 已安装时 (默认), mise 自动用 aube 安装.

```bash
mise use -g npm:prettier@latest    # 安装 prettier
mise use -g npm:snow-ai@latest     # 安装 snow-ai
# 对应配置文件中的: "npm:snow-ai" = "latest"
```

## 任务（Tasks）

```toml
# mise.toml 中定义
build = "npm run build"

[tasks.test]
run = "npm test"
description = "运行测试"
```

运行: `mise run build` 或简写 `mise build`

## 环境变量

```bash
mise set KEY=VALUE --global     # 设置全局环境变量
mise set --global --remove KEY  # 移除环境变量
```

或在 `~/.config/mise/config.toml` 的 `[env]` section 直接编辑.

> Mise 下载更新常常限流, 需要去 https://github.com/settings/tokens 获取 token (不需要添加任何权限),
> 然后导入 `GITHUB_TOKEN` 环境变量 (建议放 `~/.config/environment.d`, `GITHUB_TOKEN=xxx`).
> 或者 Fish Shell: `set -gx GITHUB_TOKEN xxx`

# Node.js 与 npm

Node.js 通过 mise 管理, 无需系统包管理器.

```bash
mise use -g node@lts   # 安装 LTS 版本 Node.js
node -v                # 确认版本
npm -v                 # 确认 npm 版本
```

## package.json 关键字段

| 字段              | 说明                                   |
| ----------------- | -------------------------------------- |
| `name`            | 包名                                   |
| `version`         | 版本号 (语义化版本 SemVer)             |
| `main`            | 入口文件 (默认 `index.js`)             |
| `type`            | `"module"` 启用 ESM, 默认 CommonJS     |
| `scripts`         | 自定义命令, 通过 `npm run <name>` 执行 |
| `dependencies`    | 生产依赖                               |
| `devDependencies` | 开发依赖 (仅开发时需要)                |
| `bin`             | CLI 命令入口                           |

## 模块系统

- **CommonJS** (默认): `const x = require('x')`, 用 `module.exports` 导出
- **ESM**: `import x from 'x'`, 用 `export default` 导出, 需在 package.json 设置 `"type": "module"`

## 项目初始化

```bash
mkdir my-project && cd my-project
npm init -y                          # 用默认值生成 package.json
npm install lodash                   # 添加依赖, 自动生成 node_modules/ 和 package-lock.json
```

项目目录结构:

```
my-project/
├── package.json         # 项目配置和依赖声明
├── package-lock.json    # 依赖版本锁定文件
├── node_modules/        # 依赖安装目录 (不要提交到 git)
└── index.js             # 入口文件
```

## npm 常用命令

| 操作         | 命令                        | 说明                                 |
| ------------ | --------------------------- | ------------------------------------ |
| 初始化项目   | `npm init -y`               | 用默认值生成 package.json            |
| 安装所有依赖 | `npm install`               | 根据 package-lock.json 安装          |
| 添加生产依赖 | `npm install lodash`        | 写入 dependencies                    |
| 添加开发依赖 | `npm install -D vitest`     | 写入 devDependencies                 |
| 移除依赖     | `npm uninstall lodash`      | 从 node_modules 和 package.json 删除 |
| 运行脚本     | `npm run build`             | 执行 package.json scripts            |
| 运行测试     | `npm test`                  | 执行 scripts.test (无参数脚本)       |
| 运行启动     | `npm start`                 | 执行 scripts.start (无参数脚本)      |
| 执行临时包   | `npx cowsay hello`          | 下载并运行一次, 不安装到项目         |
| 查看依赖树   | `npm list`                  | 树形展示已安装依赖                   |
| 查看包信息   | `npm info lodash`           | 查看远程包的版本/依赖等              |
| 审计安全漏洞 | `npm audit`                 | 检查依赖中的已知漏洞                 |
| 清除缓存     | `npm cache clean --force`   | 清理 npm 本地缓存                    |
| 全局安装     | `npm install -g typescript` | 用 `mise use -g npm:xxx` 代替        |

# aube 替代 npm

[aube](https://aube.en.dev/)（发音 `/ob/`）是 jdx 开发的快速 Node.js 包管理器 (Rust 实现), 设计为 npm/pnpm/yarn/bun 的 drop-in 替代.

```bash
mise use -g aube   # 通过 mise 安装
```

核心特性:

- **全局内容寻址存储**: 多项目共享包文件 (按 BLAKE3 哈希存储)
- **隔离 node_modules**: 只有直接依赖在顶层, 传递依赖在 `node_modules/.aube/` 下 (防止幽灵依赖)
- **lockfile 兼容**: 直接读写 `package-lock.json`, `pnpm-lock.yaml`, `yarn.lock`, `bun.lock`
- **自动安装**: 运行脚本前自动检查并安装过期依赖
- **lifecycle scripts 默认不执行**: 需要 `aube approve-builds` 显式批准 (更安全)

## npm → aube 命令对照

| npm                       | aube                             | 说明                  |
| ------------------------- | -------------------------------- | --------------------- |
| `npm install`             | `aube install`                   | 安装所有依赖          |
| `npm install lodash`      | `aube add lodash`                | 添加依赖              |
| `npm install -D vitest`   | `aube add -D vitest`             | 添加开发依赖          |
| `npm uninstall lodash`    | `aube remove lodash`             | 移除依赖              |
| `npm run build`           | `aube run build` 或 `aubr build` | 运行脚本              |
| `npm test`                | `aube test`                      | 运行测试 (自动先安装) |
| `npm start`               | `aube start`                     | 启动                  |
| `npx cowsay hi`           | `aubx cowsay hi`                 | 执行临时包            |
| `npm list`                | `aube list`                      | 查看依赖树            |
| `npm ls --depth=0`        | `aube list --depth=0`            | 只看直接依赖          |
| `npm update`              | `aube update`                    | 更新依赖              |
| `npm pack`                | `aube pack`                      | 打包                  |
| `npm publish`             | `aube publish`                   | 发布到 npm registry   |
| `npm cache clean --force` | `aube store prune`               | 清理全局存储          |

## 快捷命令

| 快捷   | 等价于     | 说明                  |
| ------ | ---------- | --------------------- |
| `aubr` | `aube run` | 运行脚本              |
| `aubx` | `aube dlx` | 执行临时包 (npx 等价) |

### aubx (aube dlx) 详细用法

将包拉取到临时环境并执行其二进制文件. 先检查当前项目 `node_modules/.bin/` 是否已有该命令, 没有才创建临时环境安装.

```bash
aubx cowsay hello                    # 执行 cowsay, 传参 hello
aubx -p @anthropic-ai/claude-code    # 指定包名 (二进制名与包名不同时用)
aubx -p prettier@3.0 -- prettier --check .  # 指定包名+版本, -- 后是传给二进制的参数
aubx -c 'cowsay hello | tr a-z A-Z'  # shell 模式, 支持管道和重定向
aubx -p cowsay -c 'cowsay hi | lolcat'  # -p 指定包, -c 用 shell 管道
```

| 参数                             | 说明                                                        |
| -------------------------------- | ----------------------------------------------------------- |
| `-p, --package <包名>`           | 指定要安装的包名 (可重复). 当二进制名与包名不同时需要此参数 |
| `-c, --shell-mode`               | 通过 `sh -c` 执行命令, 支持管道、重定向、环境变量展开       |
| `--registry <URL>`               | 指定此次执行使用的 registry 地址                            |
| `--frozen-lockfile`              | lockfile 与 package.json 不一致时报错                       |
| `--no-frozen-lockfile`           | 总是重新解析依赖, 忽略 lockfile                             |
| `--prefer-frozen-lockfile`       | lockfile 新鲜时使用, 过期时重新解析                         |
| `--fetch-timeout <ms>`           | 单次 HTTP 请求超时 (毫秒)                                   |
| `--fetch-retries <N>`            | 失败重试次数                                                |
| `--disable-global-virtual-store` | 禁用全局虚拟存储, 在项目内物化依赖                          |

## 关键差异 (vs npm)

| 方面              | npm                   | aube                          |
| ----------------- | --------------------- | ----------------------------- |
| node_modules 布局 | 扁平 (所有依赖在顶层) | 隔离 (只有直接依赖在顶层)     |
| lifecycle scripts | 所有依赖的脚本都执行  | 默认不执行, 需 approve-builds |
| 运行脚本前        | 需手动先 npm install  | 自动检测并安装过期依赖        |
| 全局存储          | 无, 每个项目独立下载  | 内容寻址存储, 多项目共享      |
| lockfile          | package-lock.json     | 兼容所有主流 lockfile         |

## 配置

```bash
aube config get registry                # 查看配置
aube config set auto-install-peers false  # 设置配置
aube config list --json                 # 列出所有配置
```

用户配置文件: `~/.config/aube/config.toml`

# uv

[uv](https://docs.astral.sh/uv/) 是 Astral 开发的 Python 包和项目管理器 (Rust 实现), 替代 pip/venv/pipx. 通过 mise 安装: `mise use -g uv`.

## 新项目

```bash
uv init my-project      # 创建项目目录, 生成 pyproject.toml, .python-version, main.py 等
cd my-project
uv add requests rich    # 添加依赖, 同时生成 uv.lock 和 .venv
uv run main.py          # 运行 (uv run 前会自动 sync)
```

## 已有 pyproject.toml 的项目

```bash
uv sync                 # 创建 .venv + 生成 uv.lock + 安装所有依赖
uv run python app.py    # 运行
```

## 已有 requirements.txt 的项目

```bash
uv venv                                  # 创建 .venv
uv pip install -r requirements.txt       # 安装依赖到 .venv
# 或者精确同步 (移除不在 requirements.txt 中的包):
uv pip sync requirements.txt
```

## 依赖管理

| 操作                | 命令                                 | 说明                               |
| ------------------- | ------------------------------------ | ---------------------------------- |
| 添加依赖            | `uv add requests`                    | 写入 pyproject.toml + 更新 uv.lock |
| 添加精确版本        | `uv add 'requests==2.31.0'`          | 锁定版本                           |
| 添加开发依赖        | `uv add -D pytest`                   | 写入 [dependency-groups] dev       |
| 移除依赖            | `uv remove requests`                 | 从 pyproject.toml 和 uv.lock 移除  |
| 安装所有依赖        | `uv sync`                            | 根据 uv.lock 安装到 .venv          |
| 升级全部            | `uv lock --upgrade`                  | 重新解析所有依赖                   |
| 升级单个包          | `uv lock --upgrade-package requests` | 只升级指定包                       |
| 安装到已激活的 venv | `uv pip install ruff`                | 不经过 pyproject.toml              |

## 运行命令

| 操作            | 命令                           | 说明                    |
| --------------- | ------------------------------ | ----------------------- |
| 运行项目        | `uv run main.py`               | 自动 sync 后执行        |
| 运行 + 传参     | `uv run -- flask run -p 3000`  | `--` 后传参             |
| 临时执行带依赖  | `uv run --with rich script.py` | 不安装, 临时下载执行    |
| 执行临时工具    | `uvx ruff check`               | 类似 pipx run, 临时环境 |
| 指定版本的工具  | `uvx ruff@0.3.0 check`         | 指定工具版本            |
| 安装工具到 PATH | `uv tool install ruff`         | 持久安装 CLI 工具       |

## 虚拟环境与 Python 版本

| 操作                 | 命令                              | 说明                 |
| -------------------- | --------------------------------- | -------------------- |
| 创建虚拟环境         | `uv venv`                         | 默认创建 .venv       |
| 指定 Python 版本     | `uv venv --python 3.12`           | 没有则自动下载       |
| 安装 Python          | `uv python install 3.12`          | 下载安装指定版本     |
| 固定项目 Python 版本 | `uv python pin 3.12`              | 写入 .python-version |
| 查看已安装版本       | `uv python list --only-installed` | 列出已安装的 Python  |

## pip 兼容模式

uv 提供 `uv pip` 子命令兼容 pip 常用工作流:

| pip                               | uv                                   | 说明             |
| --------------------------------- | ------------------------------------ | ---------------- |
| `pip install -r requirements.txt` | `uv pip install -r requirements.txt` | 安装依赖         |
| `pip install requests`            | `uv pip install requests`            | 安装单个包       |
| `pip freeze`                      | `uv pip freeze`                      | 导出已安装包列表 |
| `pip list`                        | `uv pip list`                        | 列出已安装包     |
| `python -m venv .venv`            | `uv venv`                            | 创建虚拟环境     |
