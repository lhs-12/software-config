## Mise 安装配置

```bash
curl https://mise.run | sh # 原生安装, 不用包管理器
mise trust --all ~/.config/mise # 信任stow软链接, 使配置生效

# dotfiles(stow) 涉及配置:
# 1. mise全局配置: ~/.config/mise/config.toml , 包括需要安装工具的清单和配置
# 2. shell激活配置(.bashrc 和 fish config)

mise self-update # 更新mise
mise upgrade # 安装/升级工具

# 安装工具命令示例(无需执行)
# mise install java@temurin-8 java@25 maven@3 node@lts uv@latest
# mise use -g java@25 maven@3 node@lts uv@latest
```

---

在命令行用 mise 下载更新常常 github 限流,  
需要去 [Github Token](https://github.com/settings/tokens) 获取 token(不需要添加权限),  
然后导入`GITHUB_TOKEN`环境变量.  
可以放`~/.config/environment.d/private.conf`, 格式`GITHUB_TOKEN=xxx`,  
也可以 Fish Shell 中执行 `set -gx GITHUB_TOKEN xxx`.

---

mise 支持安装 npm 的工具(用`npm:`前缀),  
用 `mise use -g` 且安装了 aube, mise 会自动用 aube 作为包管理工具

**全面使用 aube 代替 npm**:

| 场景         | npm 原用法           | 代替用法              |
| ------------ | -------------------- | --------------------- |
| 全局安装工具 | `npm install -g`     | `mise use -g npm:xxx` |
| 执行临时包   | `npx xxx`            | `aubx xxx`            |
| 项目安装依赖 | `npm install`        | `aube install`        |
| 添加依赖     | `npm install lodash` | `aube add lodash`     |
| 运行脚本     | `npm run build`      | `aubr build`          |

---

mise 可管理 python 版本, 但系统自带了 python, 且已经通过mise安装了 `uv`,  
因此不使用 mise 下载和管理 python, 都用 uv.  
mise 只用于切换到python项目目录时自动激活已存在的 python 虚拟环境(不介入依赖安装)

python(uv) 项目中的 `mise.toml` 举例:

```toml
# need to execute mise trust
[env]
PROJECT_NAME = "{{ config_root | basename }}"
_.python.venv = { path = ".venv", create = false }
[settings]
python.uv_venv_auto = "source"
```

## AI 工具配置

原理: 利用 PreToolUse 和 CLAUDE.md 之类的机制, 让工具调用准确的工具

---

**提示词**

比如 `~/.claude/CLAUDE.md`

```md
- Markdown 规则: 使用英文标点 ; 多行连续句子用两个空格换行
- Python 项目先看目录状态: 有 uv.lock 或有[project]表的pyproject.toml 用 uv(sync/add/run), 有其他lock文件用对应工具, 没有则用 uv pip 兼容层, 不要混用
- mise 管理开发工具(语言版本/依赖/code CLI/全局npm包), 会按目录自动切换版本和激活虚拟环境, 当前 shell 可能已就绪, 先确认再操作
- 全局工具用 `mise use -g pkg` 或 `mise use -g npm:pkg` 安装, 禁止 `npm install -g`
- Node.js 统一用 aube (代替 npm/npx/yarn/pnpm):
  - 安装依赖: `aube install` ; 添加: `aube add pkg` ; 移除: `aube remove pkg`
  - 运行脚本: `aubr script_name` ; 执行临时包: `aubx pkg`
  - 禁止: `npm install`, `npm uninstall`, `npm update`, `npx`, `yarn`, `pnpm`
- 系统整洁: 遵循 XDG 规范, 不在 $HOME 创建散落的 dotfiles
```

---

**PreToolUse Hook 强制执行**

Claude Code 的 `PreToolUse` Hook 在命令执行前拦截, 强制使用正确工具.  
脚本规范: 返回 exit 0 = 放行 ; 返回 exit 2 + stderr = 阻止并反馈信息

注册 Hook: `~/.claude/settings.json`

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash|Run",
        "hooks": [
          { "type": "command", "command": "python3 ~/.claude/hooks/force_tools.py", "timeout": 10 }
        ]
      }
    ]
  }
}
```

Hook 脚本: `~/.claude/hooks/force_tools.py`

```python
#!/usr/bin/env python3
"""PreToolUse hook: enforce correct tool usage"""

import json
import re
import sys

# shell token 检测: 捕获命令前的shell分隔符, 用于复合命令(如 cd x && pip install)
# 匹配: 开始符^, &&, ||, ;, 换行\n (不匹配单管道|, 语义复杂易误伤)
_SP = r"((?:^|&&|\|\||;|\n)\s*)"

# Python: 1:1 映射, 直接提示替代命令
PYTHON_DIRECT = [
    # python -m venv → uv venv
    (rf"{_SP}python3?(\.\d+)?\s+-m\s+venv\s+", "Blocked: use `uv venv` instead"),
    (rf"{_SP}python3?(\.\d+)?\s+-m\s+virtualenv\s+", "Blocked: use `uv venv` instead"),
    (rf"{_SP}virtualenv\s+", "Blocked: use `uv venv` instead"),
    # pip 只读子命令 → uv pip
    (rf"{_SP}pip3?(\.\d+)?\s+freeze\b", "Blocked: use `uv pip freeze` instead"),
    (rf"{_SP}pip3?(\.\d+)?\s+list\b", "Blocked: use `uv pip list` instead"),
    (rf"{_SP}pip3?(\.\d+)?\s+show\b", "Blocked: use `uv pip show` instead"),
    (rf"{_SP}python3?(\.\d+)?\s+-m\s+pip\s+freeze\b", "Blocked: use `uv pip freeze` instead"),
    (rf"{_SP}python3?(\.\d+)?\s+-m\s+pip\s+list\b", "Blocked: use `uv pip list` instead"),
    (rf"{_SP}python3?(\.\d+)?\s+-m\s+pip\s+show\b", "Blocked: use `uv pip show` instead"),
    # pipx → uv tool / uvx
    (rf"{_SP}pipx\s+install\s+", "Blocked: use `uv tool install` instead"),
    (rf"{_SP}pipx\s+run\s+", "Blocked: use `uvx` instead"),
    (rf"{_SP}pipx\s+uninstall\s+", "Blocked: use `uv tool uninstall` instead"),
    (rf"{_SP}pipx\s+", "Blocked: use `uv tool` instead"),
]

# Python: 场景依赖的命令，需要模型判断项目类型
PYTHON_GUIDANCE = """\
Blocked: 检查项目类型后再选命令.
判断优先级: lock文件 > pyproject.toml > requirements.txt > setup.py
有uv.lock: uv项目, 用 uv sync/add/remove/run
有poetry.lock/Pipfile.lock/pdm.lock: 用对应工具管理, 不要擅自迁移
有[project]表的pyproject.toml(无其他lock): uv sync/add/remove/run (首次自动创建.venv和uv.lock)
[project.dependencies]为空但有requirements.txt: 先 uv add -r requirements.txt
无pyproject.toml,有requirements.txt: 已有.venv?直接 uv pip install -r ; 没有? uv venv && uv pip install -r
无pyproject.toml,有setup.py: uv venv && uv pip install -e .
独立脚本: uv run script.py(无依赖) / uv run --with pkg script.py(有依赖)
临时CLI: uvx <tool> ; 全局工具: uv tool install <pkg>
原则: 不确定时不要创建环境或lock文件; 不要混用依赖管理器; uv项目不要用uv pip(会脱离lock管理)"""
PYTHON_CONTEXT = [
    # pip 系列
    rf"{_SP}pip3?(\.\d+)?\s+",
    rf"{_SP}python3?(\.\d+)?\s+-m\s+pip\s+",
    # 以下已注释: mise/venv 激活后这些命令可以直接执行, 报错了模型会自己排查
    # r"^python3?(\.\d+)?\s+(\S+\.py\b)",
    # r"^python3?(\.\d+)?\s+-c\s+",
    # r"^poetry\s+",
    # r"^pipenv\s+",
]

# Node.js 规则
NODE_RULES = [
    (rf"{_SP}yarn\s+", "Blocked: use `aube` instead of yarn"),
    (rf"{_SP}pnpm\s+", "Blocked: use `aube` instead of pnpm"),
    (rf"{_SP}npm\s+install\s+-g\s+", "Blocked: use `mise use -g npm:<pkg>` instead"),
    (rf"{_SP}npm\s+install\s+--global\s+", "Blocked: use `mise use -g npm:<pkg>` instead"),
    (rf"{_SP}npm\s+install\s+(-[A-Za-z]|--\S+)\s+(?!--)\S+", "Blocked: use `aube add` with equivalent flags instead"),
    (rf"{_SP}npm\s+install\s+--\S+", "Blocked: use `aube install` (check `aube install --help` for flag compatibility)"),
    (rf"{_SP}npm\s+update\s+", "Blocked: use `aube update` instead"),
    (rf"{_SP}npm\s+run\s+\S+", "Blocked: use `aubr <script>` instead"),
    (rf"{_SP}npm\s+(test|start|stop|restart)\b", "Blocked: use `aubr <script>` instead"),
    (rf"{_SP}npm\s+install\s+\S+", "Blocked: use `aube add <pkg>` instead"),
    (rf"{_SP}npm\s+ci\b", "Blocked: use `aube install` instead"),
    (rf"{_SP}npm\s+(uninstall|remove)\s+-g\s+", "Blocked: use `mise uninstall npm:<pkg>` instead"),
    (rf"{_SP}npm\s+(uninstall|remove)\s+", "Blocked: use `aube remove <pkg>` instead"),
    (rf"{_SP}npm\s+install\b", "Blocked: use `aube install` instead"),
    (rf"{_SP}npm\s+", "Blocked: use `aube` instead of npm"),
    (rf"{_SP}npx\s+", "Blocked: use `aubx` (check `aubx --help` for flag compatibility)"),
]


def main():
    try:
        data = json.loads(sys.stdin.read())
        if data.get("tool_name") not in ("Bash", "Run"):
            sys.exit(0)
        cmd = data.get("tool_input", {}).get("command", "").strip()
        if not cmd:
            sys.exit(0)

        # 1. Python direct blocks
        for pattern, message in PYTHON_DIRECT:
            if re.search(pattern, cmd):
                print(message, file=sys.stderr)
                sys.exit(2)

        # 2. Python guidance
        for pattern in PYTHON_CONTEXT:
            if re.search(pattern, cmd):
                print(PYTHON_GUIDANCE, file=sys.stderr)
                sys.exit(2)

        # 3. Node.js blocks
        for pattern, message in NODE_RULES:
            if re.search(pattern, cmd):
                print(message, file=sys.stderr)
                sys.exit(2)

        sys.exit(0)
    except Exception as e:
        print(f"Hook internal error: {e}", file=sys.stderr)
        sys.exit(2)


if __name__ == "__main__":
    main()
```

**Docker Compose 配置注意项**:

Docker Compose 自定义网络默认从 172.18.0.0/16 开始分配,  
与 singbox TUN 默认网段 (172.18.0.1/30) 可能冲突.  
冲突时，访问 Docker 容器服务的流量会被错误路由到 TUN 接口.

检查: `ip route show | grep 172`

解决方法:

1. docker-compose.yml 指定网段(最简单，不影响其他项目)
2. Docker daemon 配置默认网段池(全局生效)
3. 修改 singbox TUN 网段(需要客户端支持)

方法1示例:

```yml
networks:
  my-network-name:
    driver: bridge
    ipam:
      config:
        - subnet: 172.28.0.0/16
          gateway: 172.28.0.1
```
