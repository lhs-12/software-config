# Mise

## Mise 基本安装配置

```bash
curl https://mise.run | sh # 原生安装, 不用包管理器
mise trust --all ~/.config/mise # 信任stow软链接, 使配置生效

# dotfiles(stow) 涉及配置:
# 1. mise全局配置: ~/.config/mise/config.toml , 包括需要安装工具的清单和配置
# 2. shell激活配置(.bashrc 和 fish config)

mise self-update # 更新mise
mise upgrade # 安装/升级工具

# 安装工具命令示例(无需执行)
# mise install java@temurin-8 java@25 maven@3 node@lts uv@latest
# mise use -g java@25 maven@3 node@lts uv@latest
```

> 想让项目级别 `mise.toml` 生效, 需要在项目目录执行 `mise trust`

---

在命令行用 mise 下载更新常常 github 限流,  
需要去 [Github Token](https://github.com/settings/tokens) 获取 token(不需要添加权限),  
然后导入 `GITHUB_TOKEN` 环境变量.  
可以放 `~/.config/environment.d/private.conf`, 格式 `GITHUB_TOKEN=xxx`,  
也可以 Fish Shell 中执行 `set -gx GITHUB_TOKEN xxx`.

## Mise + NPM 管理

Mise 支持安装 NPM 的包(用 `npm:` 前缀),  
当使用 Mise 安装了 aube, 然后执行 `mise use -g npm:xxx`, Mise 会自动用 aube 作为 NPM 包管理工具

**全面使用 aube 代替 npm**:

| 场景         | npm 原用法           | 代替用法              |
| ------------ | -------------------- | --------------------- |
| 全局安装工具 | `npm install -g`     | `mise use -g npm:xxx` |
| 执行临时包   | `npx xxx`            | `aubx xxx`            |
| 项目安装依赖 | `npm install`        | `aube install`        |
| 添加依赖     | `npm install lodash` | `aube add lodash`     |
| 运行脚本     | `npm run build`      | `aubr build`          |

## Mise + Python 管理

虽然 Mise 可管理 Python 版本, 但系统自带 `/usr/bin/python`, 另外我们已通过 Mise 安装了 uv

Mise + Python 使用原则:

- 系统层面: 永远使用系统 Python, Mise 全局配置中不安装任何 Python 版本
- 项目层面: 所有项目 Python 版本和虚拟环境都由 uv 安装管理
- Mise作用: 仅在进入目录时自动激活已有的虚拟环境(不参与 Python 安装和依赖管理)

使用注意:

- 无论什么项目, `mise.toml` 的 `[tools]` 里都不要添加 `python = "..."`
- 避免同时设置 `_.python.venv` 与 `python.uv_venv_auto`, 两套是独立的机制, 选一套用即可

### uv 项目

适用于已用 `uv init` 或 `uv sync` 管理, 存在 `uv.lock` 的项目

项目级 `mise.toml` 示例:

```toml
[env]
PROJECT_NAME = "{{ config_root | basename }}"
[settings]
# 激活uv虚拟环境配置"source": 如果已有.venv则激活 ; "create|source": 自动创建且激活
python.uv_venv_auto = "source"
```

### 非 uv 项目 -> 转为 uv 项目

```bash
cd project-dir
uv init # 生成 pyproject.toml 和 uv.lock
uv sync # 读取 .python-version , 创建 .venv , 安装依赖
# 最后使用与上面 uv 项目一样的 mise.toml
```

### 非 uv 项目 -> 项目结构不变, uv 命令管理

> 如果只是临时运行单个脚本, 不需要长期维护的虚拟环境，可以用 `uv run python main.py`  
> `uv run` 会自动创建临时隔离环境执行脚本，退出后不留 `.venv`，也不需要 mise 配置  
> 若脚本有依赖，可使用 `uv run --with <pkg>` 或 `--with-requirements requirements.txt` 临时安装  
> 也可在脚本顶部用 PEP 723 声明依赖，`uv run` 会自动处理

建立虚拟环境:

```bash
uv venv --python 3.12 .venv        # 创建指定版本虚拟环境
uv pip install -r requirements.txt # 安装依赖
```

`mise.toml`

```toml
[env]
PROJECT_NAME = "{{ config_root | basename }}"
_.python.venv = { path = ".venv", create = false } # 仅激活, 不创建
```

网络功能测试

```bash
# netshoot: 自带大量网络调试命令的容器镜像
docker run --rm -it nicolaka/netshoot
# --rm: 容器退出后删除 ; --it: 保持stdin输入+分配终端=进入交互式shell
# 命令通过--network参数指定各种网络模式, 比如: --network host
# 查看容器网络模式: docker inspect [容器名称] | grep NetworkMode

# 容器内常用测试命令：
# ip a                           # 查看ip配置
# ip route                       # 查看路由配置
# ping 1.1.1.1                   # 测试ip连接
# dig google.com                 # 测试DNS
# curl -I https://www.google.com # 测试网络连接

# ===代理(docker默认网络模式)===
# 1. 系统代理
# 代理软件要配置"允许局域网的连接"(监听0.0.0.0)
# docker run 命令增加 --add-host=host.docker.internal:host-gateway 参数, 用于访问宿主机
# env | grep -i proxy # 查看容器内代理环境变量
# curl -x http://host.docker.internal:10808 https://www.google.com
# ss -lntp | grep 10808             # 宿主机执行: 查看代理端口是否监听
# nc -zv host.docker.internal 10808 # 容器内执行: 测试能否连接宿主机代理端口
#
# 2. Tun代理: v2rayN的Tun(singbox)已经配置Docker网段的流量走代理
```

---

**Docker Compose 配置注意项**:

Docker Compose 自定义网络默认从 `172.18.0.0/16` 开始分配,  
与 singbox TUN 默认网段 (`172.18.0.1/30`) 可能冲突.  
冲突时，访问 Docker 容器服务的流量会被错误路由到 TUN 接口.

检查: `ip route show | grep 172`

解决方法:

1. docker-compose.yml 指定网段(最简单，不影响其他项目)
2. Docker daemon 配置默认网段池(全局生效)
3. 修改 singbox TUN 网段(需要客户端支持)

方法1示例:

```yml
networks:
  my-network-name:
    driver: bridge
    ipam:
      config:
        - subnet: 172.28.0.0/16
          gateway: 172.28.0.1
```
