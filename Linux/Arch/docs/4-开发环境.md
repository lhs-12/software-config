## Mise 安装配置

```bash
curl https://mise.run | sh # 原生安装, 不用包管理器
mise trust --all ~/.config/mise # 信任stow软链接, 使配置生效

# dotfiles(stow) 涉及配置:
# 1. mise全局配置: ~/.config/mise/config.toml , 包括需要安装工具的清单和配置
# 2. shell激活配置(.bashrc 和 fish config)

mise self-update # 更新mise
mise upgrade # 安装/升级工具

# 安装工具命令示例(无需执行)
# mise install java@temurin-8 java@25 maven@3 node@lts uv@latest
# mise use -g java@25 maven@3 node@lts uv@latest
```

---

在命令行用 mise 下载更新常常 github 限流,  
需要去 [Github Token](https://github.com/settings/tokens) 获取 token(不需要添加权限),  
然后导入`GITHUB_TOKEN`环境变量.  
可以放`~/.config/environment.d/private.conf`, 格式`GITHUB_TOKEN=xxx`,  
也可以 Fish Shell 中执行 `set -gx GITHUB_TOKEN xxx`.

---

mise 支持安装 npm 的工具(用`npm:`前缀),  
用 `mise use -g` 且安装了 aube, mise 会自动用 aube 作为包管理工具

**全面使用 aube 代替 npm**:

| 场景         | npm 原用法           | 代替用法              |
| ------------ | -------------------- | --------------------- |
| 全局安装工具 | `npm install -g`     | `mise use -g npm:xxx` |
| 执行临时包   | `npx xxx`            | `aubx xxx`            |
| 项目安装依赖 | `npm install`        | `aube install`        |
| 添加依赖     | `npm install lodash` | `aube add lodash`     |
| 运行脚本     | `npm run build`      | `aubr build`          |

---

mise 可管理 python 版本, 但系统自带了 python, 且已经通过mise安装了 `uv`,  
因此不使用 mise 下载和管理 python, 都用 uv.  
mise 只用于切换到python项目目录时自动激活已存在的 python 虚拟环境(不介入依赖安装)

python(uv) 项目中的 `mise.toml` 举例:

```toml
# need to execute mise trust
[env]
PROJECT_NAME = "{{ config_root | basename }}"
_.python.venv = { path = ".venv", create = false }
[settings]
python.uv_venv_auto = "source"
```

## AI 工具配置

原理: 利用 PreToolUse 和 CLAUDE.md 之类的机制, 让工具调用准确的工具

---

**提示词**

比如 `~/.claude/CLAUDE.md`

```md
- Markdown 多行连续句子用两个空格换行
- Python 统一用 uv (代替 pip/venv/pipx/pipenv/poetry):
  - 创建项目: `uv init`
  - 添加依赖: `uv add pkg`
  - 移除依赖: `uv remove pkg`
  - 安装依赖: `uv sync`
  - 运行命令: `uv run python app.py`
  - 临时执行带依赖的脚本: `uv run --with pkg script.py`
  - 执行临时工具: `uvx tool` (类似 pipx run)
  - 安装全局工具: `uv tool install pkg` (类似 pipx install)
  - 已有 requirements.txt 的项目: 迁移到 uv 管理用 `uv add -r requirements.txt` (写入 pyproject.toml+uv.lock) ; 仅安装不迁移用 `uv venv && uv pip install -r requirements.txt`
  - 已有 pyproject.toml 的项目: `uv sync`
  - 不要 `pip install`, `pip uninstall`, `python -m pip`, `python -m venv`, `pipx`, `pipenv`, `poetry`, `virtualenv`
- Node.js 统一用新工具 aube (代替 npm/npx/yarn/pnpm):
  - 安装依赖: `aube install`
  - 添加依赖: `aube add pkg`
  - 添加开发依赖: `aube add -D pkg`
  - 移除依赖: `aube remove pkg`
  - 运行脚本: `aubr script_name`
  - 执行临时包: `aubx pkg`
  - 不要 `npm install`, `npm uninstall`, `npm update`, `npx`, `yarn`, `pnpm`
- 全局工具用 `mise use -g pkg` 或 `mise use -g npm:pkg` 安装, 禁止 `npm install -g`
- mise 管理开发相关工具: 编程语言/版本依赖管理, code CLI, 全局 npm 包
- 系统整洁: 遵循 XDG 规范, 不在 $HOME 创建散落的 dotfiles
```

**PreToolUse Hook 强制执行**

Claude Code 的 `PreToolUse` Hook 在命令执行前拦截, 强制使用正确工具.  
脚本规范: 返回 exit 0 = 放行 ; 返回 exit 2 + stderr = 阻止并反馈信息

注册 Hook: `~/.claude/settings.json`

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash|Run",
        "hooks": [
          { "type": "command", "command": "python3 ~/.claude/hooks/force_tools.py", "timeout": 10 }
        ]
      }
    ]
  }
}
```

Hook 脚本: `~/.claude/hooks/force_tools.py`

```python
#!/usr/bin/env python3
"""PreToolUse hook: enforce correct tool usage"""

import json
import re
import sys

RULES = [
    # Python: pip → uv
    (r"^pip3?(\.\d+)?\s+install\s+-r\s+", "uv pip install -r ", None),
    (r"^pip3?(\.\d+)?\s+install\s+-c\s+", "uv pip install -c ", None),
    (r"^pip3?(\.\d+)?\s+install\s+", "uv add ", None),
    (r"^pip3?(\.\d+)?\s+uninstall\s+", "uv remove ", None),
    (r"^pip3?(\.\d+)?\s+", "uv pip ", None),
    # Python: python -m <module> → uv
    (r"^python3?(\.\d+)?\s+-m\s+pip\s+install\s+-r\s+", "uv pip install -r ", None),
    (r"^python3?(\.\d+)?\s+-m\s+pip\s+install\s+-c\s+", "uv pip install -c ", None),
    (r"^python3?(\.\d+)?\s+-m\s+pip\s+install\s+", "uv add ", None),
    (r"^python3?(\.\d+)?\s+-m\s+pip\s+", "uv pip ", None),
    (r"^python3?(\.\d+)?\s+-m\s+venv\s+", None, "Blocked: use `uv venv` instead"),
    (r"^python3?(\.\d+)?\s+-m\s+virtualenv\s+", None, "Blocked: use `uv venv` instead"),
    (r"^virtualenv\s+", None, "Blocked: use `uv venv` instead"),
    (r"^python3?(\.\d+)?\s+", "uv run python ", None),
    # Python: pipx/pipenv/poetry → uv
    (r"^pipx\s+install\s+", "uv tool install ", None),
    (r"^pipx\s+run\s+", "uvx ", None),
    (r"^pipx\s+uninstall\s+", "uv tool uninstall ", None),
    (r"^pipx\s+", "uv tool ", None),
    (r"^pipenv\s+", None, "Blocked: use `uv` instead of pipenv"),
    (r"^poetry\s+", None, "Blocked: use `uv` instead of poetry"),
    # Python tools → uv run <tool>
    (r"^((?:pytest|ruff|mypy|black|flake8|isort|pylint|bandit|safety)\b)", "uv run \\1", None),
    # Node.js: yarn/pnpm → aube
    (r"^yarn\s+", None, "Blocked: use `aube` instead of yarn"),
    (r"^pnpm\s+", None, "Blocked: use `aube` instead of pnpm"),
    # Node.js: npm/npx → aube/aubx
    (r"^npm\s+install\s+-g\s+", None, "Blocked: use `mise use -g npm:<pkg>` instead"),
    (r"^npm\s+update\s+", None, "Blocked: use `aube update` instead"),
    (r"^npm\s+run\s+\S+", None, "Blocked: use `aubr <script>` instead"),
    (r"^npm\s+(test|start|stop|restart)\b", None, "Blocked: use `aubr <script>` instead"),
    (r"^npm\s+install\s+\S+", None, "Blocked: use `aube add <pkg>` instead"),
    (r"^npm\s+ci\b", None, "Blocked: use `aube install` instead"),
    (r"^npm\s+(uninstall|remove)\s+", None, "Blocked: use `aube remove <pkg>` instead"),
    (r"^npm\s+install\b", "aube install", None),
    (r"^npm\s+", None, "Blocked: this project uses aube instead of npm"),
    (r"^npx\s+", "aubx ", None),
]


def main():
    try:
        data = json.loads(sys.stdin.read())
        if data.get("tool_name") not in ("Bash", "Run"):
            sys.exit(0)
        cmd = data.get("tool_input", {}).get("command", "").strip()
        if not cmd:
            sys.exit(0)
        for pattern, replacement, message in RULES:
            if re.match(pattern, cmd):
                if replacement:
                    new_cmd = re.sub(pattern, replacement, cmd)
                    print(f"Blocked. Try: {new_cmd}", file=sys.stderr)
                else:
                    print(message, file=sys.stderr)
                sys.exit(2)
        sys.exit(0)
    except Exception:
        sys.exit(0)


if __name__ == "__main__":
    main()
```

**Docker Compose 配置注意项**:

Docker Compose 自定义网络默认从 172.18.0.0/16 开始分配,  
与 singbox TUN 默认网段 (172.18.0.1/30) 可能冲突.  
冲突时，访问 Docker 容器服务的流量会被错误路由到 TUN 接口.

检查: `ip route show | grep 172`

解决方法:

1. docker-compose.yml 指定网段(最简单，不影响其他项目)
2. Docker daemon 配置默认网段池(全局生效)
3. 修改 singbox TUN 网段(需要客户端支持)

方法1示例:

```yml
networks:
  my-network-name:
    driver: bridge
    ipam:
      config:
        - subnet: 172.28.0.0/16
          gateway: 172.28.0.1
```
