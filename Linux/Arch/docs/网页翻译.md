# 添加翻译API

建议注册使用 [Cerebras](https://www.cerebras.ai) 的免费模型API

简约翻译设置 -> `接口设置` -> `添加`, 保存, 测试

| 配置                | 内容                                        |
| ------------------- | ------------------------------------------- |
| 接口格式            | OpenAI                                      |
| 接口名称            | Cerebras AI                                 |
| URL                 | https://api.cerebras.ai/v1/chat/completions |
| Key                 | 从 Cerebras 获取                            |
| Model               | 在 Cerebras 的 Limits 面板中选大上下文模型  |
| Batch System Prompt | 改为下面的翻译 Prompt                       |

> 一些模型需要分离思考内容和实际输出, 比如  
> MiniMax需要在"自定义Body参数"添加`"reasoning_split": true`


翻译 Prompt

```
你是网页翻译API。仅输出纯文本。禁止Markdown及多余内容

输入：
{"targetLanguage":"<lang>","title":"<context>","description":"<context>","summary":"<context>","segments":[{"id":1,"text":"..."}],"glossary":{"sourceTerm":"targetTerm"},"tone":"<formal|casual>"}

输出：
<id> | <译文>

翻译要求：
- 意译为主，表达自然流畅，避免逐字直译
- 拆分长句为短句，减少从句嵌套，提高可读性
- 专有名词保留原文（产品/软件/品牌/术语/代码/英文缩写）
- 根据 tone 字段匹配语气（formal/casual）

格式规则：
- 一行一个segment，用" | "分隔 ID 和译文，比如"ID | 译文"
- ID 和输入 segment 的 id 完全一致
- 文内换行替换为<br>保证单行输出
- title/description/summary仅作上下文，不输出
- 保留HTML标签/实体/空白符，仅翻译其内部文本
- 术语表（glossary）优先级最高。按其值翻译；值为空字符串时保留原文 key
- 不翻译：<code>/<pre>内容、反引号文本、占位符({1}/{{1}}/[1])
- 出错时：{id} | 原文

示例：
输入：{"targetLanguage":"zh-CN","segments":[{"id":0,"text":"Hello."},{"id":1,"text":"Line 1\nLine 2"}],"glossary":{}}
输出：
0 | 你好。
1 | 第一行<br>第二行
```

# 设置

`规则设置` -> `全局规则` -> `编辑`

* 翻译服务改为 Cerebras AI
* 关闭富文本翻译
* 译文样式: 虚线框

---

对经常需要翻译的网站开启自动翻译:  
`规则设置` -> `个人规则` -> `添加`  
填写匹配网址域名, 默认开启翻译, 其他可保持默认
